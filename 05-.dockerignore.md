# `.dockerignore`


### What is `.dockerignore`?

* `.dockerignore` tells Docker **which files/folders to ignore**
* Ignored files are **not copied** into the Docker image
* Works similar to **`.gitignore`**
* Helps to:

  * Reduce image size
  * Speed up build
  * Avoid unnecessary files

---

### Why `.dockerignore` is Important

* Prevents copying:

  * `node_modules` (rebuilt inside container)
  * Logs and cache files
  * `.git` folder
* Makes Docker builds **clean and faster**

---

### Common `.dockerignore` for Node.js App

```dockerignore
# Node dependencies (install inside container)
node_modules

# Ignoring multiple files with same extension
*.txt
*.ts

# npm / yarn logs
npm-debug.log
yarn-error.log

# Environment files
.env

# Git files
.git
.gitignore

# OS / editor files
.DS_Store
.vscode
.idea

# Docker files (optional)
Dockerfile
docker-compose.yml
```

---

### How `.dockerignore` Works

* Docker reads `.dockerignore` **before** `COPY . .`
* All listed files are **excluded from the build context**
* Only required files are sent to Docker daemon

---

### With vs Without `.dockerignore`

**Without `.dockerignore`**

* Copies `node_modules`
* Larger image
* Slower build

**With `.dockerignore`**

* Clean image
* Faster build
* Fewer issues

---

### Exam-Friendly One Line

> **`.dockerignore` is used to exclude unnecessary files from being copied into a Docker image during build time.**

---

### Very Important Tip

* Always use `.dockerignore` in **Node.js projects**
* Especially to ignore `node_modules`
